"Exalead Cloudview":http://www.exalead.com/software/products/cloudview/ 5.1 is delivered with an "Eclipse":http://www.eclipse.org/ project for the Search UI Application Interface, this application is build on "Spring":http://www.springsource.org/ 2.4.4.

h3. The Problem

When requesting the page http://host:port/search-ui-framework/search/C.view=CloudView-50/results?q=test I got the following error (see the full stack trace at the end of the post):


{% highlight java %}
Error evaluating XPath expression "string($answer/Answer/@context)": javax.xml.transform.TransformerException: Unknown error in XPath. 
{% endhighlight %}

h3. The Cause

The following line of code in WEB-INF\views\jsp\search\results.jsp is raising the error

{% highlight java %}
<x:set var="searchContext" select="string($answer/Answer/@context)"/>
{% endhighlight %}

bq. This seems to be caused by a slightly different implementation of the "java.beans.PropertyDescriptor" class in IBM's JDK class libraries... breaking an assumption in Spring's GenericTypeAwarePropertyDescriptor subclass... ??Juergen Hoeller, "Co-founder of the Spring Framework":http://www.springsource.com/people/jhoeller?? 

h3. The Solution

Removing the string() solve this issue.

{% highlight java %}
<x:set var="searchContext" select="$answer/Answer/@context"/>
{% endhighlight %}
